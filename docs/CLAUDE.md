# docs/

Long-form docs. Everything lives under `plans/`; pattern is one design doc + one implementation plan per feature, dated `YYYY-MM-DD-<topic>[-design].md`.

## Subdirectories

| Directory | What                                    | When to read                                                              |
| --------- | --------------------------------------- | ------------------------------------------------------------------------- |
| `plans/`  | Design docs + implementation plans      | Reviewing design rationale, writing a new plan, recovering context        |
